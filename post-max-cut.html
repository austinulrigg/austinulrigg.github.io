Maximum Cut and the Goemans–Williamson Algorithm

Download the original paper (PDF)

Given a graph G(V,E)G(V,E)G(V,E) — which I'll just call GGG — consider partitioning the vertex set VVV into two disjoint sets AAA and BBB, and counting the number of edges ek=(vi,vj)e_k=(v_i,v_j)ek​=(vi​,vj​) where vi∈Av_i\in Avi​∈A and vj∈Bv_j\in Bvj​∈B.

For this partition we have two edges crossing between AAA and BBB. Is it possible to have more? For an arbitrary graph GGG, can we maximize the number of edges that cross from AAA to BBB, and identify the partition(s) that attain it? This is the maximum cut problem. The size of a cut is the number of edges crossing between AAA and BBB, and the maximum cut is a cut with at least as many edges as every other cut.

An NP-complete problem

The maximum cut problem is known to be NP-complete — in fact, it was one of Karp's original 21 NP-complete problems. Because solving any one NP-complete problem with a polynomial-time algorithm would settle the famous P=NPP = NPP=NP problem, one of the Clay Institute's millennium prize problems, we don't expect an efficient algorithm that always finds the maximum cut exactly. Instead we look for lower bounds and approximation algorithms.

A quadratic reformulation

A cut for free

It is easy to find a cut of any graph that attains at least half of the max cut. Writing #E(A,B)\#E(A,B)#E(A,B) for the number of edges in the cut of a partition V=A∪BV=A\cup BV=A∪B, there is always a partition with

#E(A,B) ≥ maxcut⁡(G)2.\#E(A,B)\ \ge\ \frac{\operatorname{maxcut}(G)}{2}.#E(A,B) ≥ 2maxcut(G)​.

A random split already achieves this in expectation — each edge crosses with probability 12\tfrac1221​. The Edwards–Erdős bound sharpens it: every graph with mmm edges has a cut of size at least

and this is tight for complete graphs. So half the edges is a floor we never have to work for.

Max cut on special graphs

For some families the maximum cut is known exactly. A complete graph on 2m2m2m vertices has maximum cut m2m^2m2, attained by splitting the vertices as evenly as possible; the proof is the simple inequality m2>j(2m−j)m^2 > j(2m-j)m2>j(2m−j) for any uneven split. And a tree with mmm edges has maximum cut exactly mmm — a tree has no cycles, so it is bipartite, and the two-coloring puts every edge across the cut.

The Goemans–Williamson algorithm

To do better in general, we relax the quadratic program. Instead of forcing each xi∈{−1,1}x_i\in\{-1,1\}xi​∈{−1,1}, let each vertex be a unit vector and maximize

SDP⁡(G)=14max⁡∥xi∥=1∑i,j=1naij (1−⟨xi,xj⟩) ≥ maxcut⁡(G).\operatorname{SDP}(G)=\frac14\max_{\lVert x_i\rVert=1}\sum_{i,j=1}^{n} a_{ij}\,(1-\langle x_i,x_j\rangle)\ \ge\ \operatorname{maxcut}(G).SDP(G)=41​∥xi​∥=1max​i,j=1∑n​aij​(1−⟨xi​,xj​⟩) ≥ maxcut(G).

This is a semidefinite program, and unlike the original it can be solved in polynomial time (for instance with interior-point or ellipsoid methods). The inequality holds because the ±1\pm 1±1 labels are just the one-dimensional special case.

But a solution to the semidefinite program is a list of nnn vectors in Rn\mathbb{R}^nRn, not a cut of our graph. Surprisingly, the most difficult part of using this relaxation is translating the result back into a meaningful cut. The key idea is randomized hyperplane rounding: round each vector to 111 or −1-1−1 depending on which side of a random hyperplane it lands on.

To choose the hyperplane, sample nnn independent standard normal variables y1,…,yny_1,\dots,y_ny1​,…,yn​ and normalize n⃗=(y1,…,yn)/∥(y1,…,yn)∥\vec n = (y_1,\dots,y_n)/\lVert(y_1,\dots,y_n)\rVertn=(y1​,…,yn​)/∥(y1​,…,yn​)∥ to get its normal direction. Two vectors the program pushed far apart — the ones it "wants" cut — are the ones most likely to be separated.

The whole argument rests on one clean fact: an edge (i,j)(i,j)(i,j) ends up in the cut with probability θ/π\theta/\piθ/π, where θ\thetaθ is the angle between xix_ixi​ and xjx_jxj​, and θπ ≥ 0.878 (1−cos⁡θ)for all θ∈[0,π].\frac{\theta}{\pi}\ \ge\ 0.878\,\bigl(1-\cos\theta\bigr)\qquad\text{for all }\theta\in[0,\pi].πθ​ ≥ 0.878(1−cosθ)for all θ∈[0,π].

So the algorithm always lands within about 88%88\%88% of the best possible cut — a hard guarantee, on every graph, in polynomial time. Under the unique games conjecture no efficient algorithm can beat this constant, which ties 0.8780.8780.878 back to the P=NPP = NPP=NP question it started from.

Max cut turns up across theoretical computer science, network design, machine learning, and even theoretical physics, and the tools behind it — semidefinite programming, randomized rounding, and the probabilistic method — reach much further than this one problem.
